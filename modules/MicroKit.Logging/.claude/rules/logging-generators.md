# Rule: Logging Generators

Rules for `MicroKit.Logging.Generators` — Roslyn source generators.

- **`IIncrementalGenerator` only** — `ISourceGenerator` is legacy
- **Incremental pipeline** — all data via `IncrementalValueProvider`, never full `Compilation` walk
- **Deterministic output** — same input always produces identical generated code (required for Roslyn caching)
- **Generated filenames** — `{TypeName}.g.cs` convention
- **No diagnostics in generators** — errors go through `MicroKit.Logging.Analyzers`
- **No `// <auto-generated>` modification** — use the standard Roslyn header via `SourceProductionContext`
